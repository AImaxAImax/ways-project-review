# WAYS Plate QC Human Approval Dashboard Contract

This contract implements Gate 2 from `WAYS_KANBAN_AND_QC_GATES.md`: every generated still plate stops in Plate QC until Josh gives a human approve/deny decision and confirms the render lane. Automated VLM output is advisory evidence only; it never advances a plate without human persistence.

## Non-negotiables

1. One row/card per plate, not per video.
2. Show both full-size preview and phone-size preview.
3. Show a visible caption-safe-zone cue on the phone-size preview.
4. Show local Ollama `gemma3:12b` VLM criteria and scores as advisory.
5. Surface detected text, labels, numbers, logos, UI, watermarks as hard-blocker evidence.
6. Suggest animation friendliness and render lane, but require Josh to confirm the lane.
7. Persist Josh's approve/deny and confirmed lane before any render job may consume the plate.
8. Do not treat the VLM score as final authority. Human decision is the gate state.

## Output / persistence

The dashboard writes `dashboard/plate_qc_decisions.json`. Render workers read only this persisted file, not browser localStorage and not the advisory VLM score.

```json
{
  "version": 1,
  "updated_at": "2026-06-06T00:12:31Z",
  "decisions": {
    "mantis_shrimp__shot03__hf_ultra__v01": {
      "plate_id": "mantis_shrimp__shot03__hf_ultra__v01",
      "decision": "approved",
      "reviewer": "Josh",
      "reviewed_at": "2026-06-06T00:12:31Z",
      "confirmed_lane": "wan_i2v",
      "human_score": 9.0,
      "notes": "Approved if motion stays lateral and calm.",
      "vlm_was_advisory_only": true,
      "override_hard_blockers": false
    }
  }
}
```

Allowed `decision` values: `approved`, `denied`.

Allowed `confirmed_lane` values: `wan_i2v`, `motion_graphic`, `still_motion`, `regenerate`, `hold`.

Render may start only when:

```text
decision == approved
confirmed_lane in {wan_i2v, motion_graphic, still_motion}
vlm_was_advisory_only == true
reviewer == Josh
reviewed_at is present
```

If a plate has `detected_hard_blockers` and Josh still approves, the dashboard must require `override_hard_blockers: true` and a note explaining the false positive or accepted exception. This preserves the “hard blocker” evidence without making the VLM final authority.

## UI requirements

Each plate card must include:

- Header: slug, shot id, plate id, generator, created time.
- Full-size preview: click/open original still at native size.
- Phone-size preview: 9:16 scaled mock phone frame with caption-safe-zone overlay.
- Beat text / intended spoken claim so Josh can judge say-dog-see-dog.
- VLM advisory panel:
  - model name and run time;
  - overall advisory score;
  - per-criterion scores;
  - advisory summary;
  - explicit banner: `VLM advisory only — Josh approval controls Gate 2`.
- Hard-blocker panel:
  - detected text/logo/UI/watermark entries;
  - red warning if any exist;
  - override-note requirement if approved anyway.
- Animation/lane panel:
  - animation friendliness score;
  - texture density;
  - boundary clarity;
  - expected failure modes;
  - suggested lane;
  - lane-confirm dropdown.
- Human controls:
  - approve;
  - deny/regenerate;
  - confirmed lane;
  - optional human score;
  - notes;
  - save button;
  - persisted status with timestamp.

## `gemma3:12b` VLM prompt contract

Use Ollama locally. Recommended command shape for a worker that creates the queue file:

```bash
ollama run gemma3:12b '<prompt with image context from your vision wrapper>'
```

The model response must be coerced into this JSON shape before writing `plate_qc_queue.json`:

```json
{
  "overall_advisory_score": 0,
  "scores": {
    "no_text_logo_ui": 0,
    "caption_safe_negative_space": 0,
    "phone_size_concept_read": 0,
    "realism_anatomy_family_tone": 0,
    "animation_friendliness": 0,
    "lane_fit_wan_i2v": 0,
    "lane_fit_motion_graphic": 0
  },
  "detected_hard_blockers": [
    {
      "type": "text|logo|ui|watermark|number|label",
      "location": "short visible location",
      "evidence": "what appears to be there",
      "confidence": 0.0
    }
  ],
  "caption_safe_zone": {
    "present": true,
    "recommended_anchor": "upper_third|middle|lower_third|none",
    "notes": "short note"
  },
  "animation": {
    "friendliness_score": 0,
    "boundary_clarity": 0,
    "texture_density": 0,
    "expected_failure_modes": [],
    "suggested_lane": "wan_i2v|motion_graphic|still_motion|regenerate|hold",
    "rationale": "short note"
  },
  "advisory_summary": "one paragraph"
}
```

Prompt criteria:

- Be harsh about any visible text, labels, numbers, logos, UI, or watermarks.
- Judge phone-size concept read: if the viewer cannot instantly understand the subject/claim on a phone, score low.
- Judge caption-safe negative space: there must be room for large centered creator-style captions without covering the subject.
- For animals/nature, realism and anatomy come before softness/family tone.
- Route strong-prior animals/recognizable objects toward `wan_i2v` only when anatomy, boundaries, and texture density can survive motion.
- Route process/concept/diagram beats toward `motion_graphic` when photoreal I2V is likely to shimmer, morph, or fail the claim.
- Never output an approval. Output only advisory evidence.

## API contract for the provided local server

A no-dependency reference server is provided at `scripts/plate_qc_dashboard_server.py`.

Run:

```bash
cd C:\dev\curious-shorts
python scripts\plate_qc_dashboard_server.py --queue dashboard\plate_qc_queue.json --decisions dashboard\plate_qc_decisions.json --port 8765
```

WSL equivalent:

```bash
cd /mnt/c/dev/curious-shorts
python3 scripts/plate_qc_dashboard_server.py --queue dashboard/plate_qc_queue.json --decisions dashboard/plate_qc_decisions.json --port 8765
```

Endpoints:

- `GET /` — dashboard UI.
- `GET /api/queue` — queue plus existing decisions.
- `POST /api/decision` — persist one Josh decision.
- `GET /media/<relative-path>` — serve a preview image from project root.

`POST /api/decision` body:

```json
{
  "plate_id": "mantis_shrimp__shot03__hf_ultra__v01",
  "decision": "approved",
  "reviewer": "Josh",
  "confirmed_lane": "wan_i2v",
  "human_score": 9,
  "notes": "Approved.",
  "override_hard_blockers": false
}
```

Server validation:

- Unknown plate id rejected.
- Decision outside `approved|denied` rejected.
- Lane outside the allowed lane set rejected.
- Approval without renderable lane rejected.
- Approval over detected hard blockers rejected unless `override_hard_blockers` is true and notes are non-empty.
- All saved decisions add `reviewed_at`, `vlm_was_advisory_only: true`, and `schema_version: 1`.

## Downstream render contract

Render workers must build `lane_map.json` from `plate_qc_decisions.json`, not from VLM suggestions:

```json
{
  "shot03": {
    "plate_id": "mantis_shrimp__shot03__hf_ultra__v01",
    "image_path": "test_cases/mantis_shrimp/assets/plate_generation/shot03_hf_ultra_v01.png",
    "lane": "wan_i2v",
    "approved_by": "Josh",
    "approved_at": "2026-06-06T00:12:31Z"
  }
}
```

Denied plates move back to single-plate regeneration. Do not rebuild the entire video unless Josh explicitly rejects the concept.
